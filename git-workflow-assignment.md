# Git Workflow Assignment

This is a group assignment.

Your team must design and agree on a Git workflow for your software project. The purpose of this task is to make your development process clear, consistent, and realistic for your team.

## Task

Write a short report that explains the Git workflow your team will use during the course.

The report should describe:

- your branching strategy
- how new work is started and merged
- how pull requests are handled
- what kind of commit message style your team will follow
- why this workflow fits your project and team

Do not use AI tools to write the report. Only AI tool allowed is Actual Intelligence, your own brain.

## What to Include

Your report must include the following:

### 1. Team Workflow Choice

- Name and describe the workflow you will use
- Examples: feature branches, GitHub flow, simplified Git flow, trunk-based development
- Link to your GitHub repository

### 2. Branching Strategy

- What is your main branch called?
- Will you use feature branches, release branches, or both?
- Who can merge to the main branch?

### 3. Pull Request Process

- When is a pull request required?
- Who reviews pull requests?
- What must be checked before merging?

### 4. Commit Message Guidelines

- Explain the general style of your commit messages
- Give 2-3 example commit messages

### 5. Justification

- Explain why this workflow is a good fit for:
  - your team size
  - your project scope
  - your course timeline
  - your way of working

## Minimum Requirements

To pass this assignment:

- the workflow is clearly described
- the team agreement is realistic and usable
- commit message guidelines are included
- reasons for the chosen workflow are explained
- the report is written in clear English

## Recommended Length

1-2 pages.
