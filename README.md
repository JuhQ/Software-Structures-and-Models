# Software Structures & Models - Weekly Plan

## Overview

| Week | Main Goal                  | Main Output                                      |
| ---- | -------------------------- | ------------------------------------------------ |
| 1    | Team + topic + planning    | Group formed + project plan + feasibility, tools |
| 2    | Implementation kickoff     | Repo active + initial codebase + DB connection   |
| 3    | Development sprint         | Core features and database implementation        |
| 4    | CI/CD start + midterm demo | Mandatory midterm demo completed                 |
| 5    | Development sprint         | CI/CD deadline completed + feature progress      |
| 6    | Development + polish       | Project cleanup and documentation progress       |
| 7    | Final prep                 | Presentation material submitted                  |
| 8    | Delivery                   | Presentation + final project submission          |

## Quick Rules

- Attendance is mandatory (minimum 80%).
- If attendance is too low, the project grade is 1.
- Midterm demo at the course midpoint (Week 4) is mandatory.
- Final project submission and presentation are mandatory.
- Classes start on the hour.
- Attendance system: attend.metropolia.fi

**NOTE:**
Course is 15 ECTS. 1 ECTS = 27 hours of work, total expected workload is around **405 hours per student**. This includes all project work, meetings, and course requirements.

## Schedule and Deliverables

### Week 1

Focus:

- Course intro, team formation, and project topics
  - Each team can choose their own project topic, teachers can also provide project ideas if needed
- Basics of software structures, Agile, and sprint workflow
- Project planning, feasibility study, vision, and design
- Database planning

Do this week:

- Form your team (3-4 people)
  - Create discord, whatsapp, or other communication channel
  - Pick your topic
    - Deadline wednesday afternoon
  - Pick product coordinator (responsible for project management)
    - This person is also responsible for contacting teachers if needed and ensuring project deadlines are met
  - Create team Drive + planning docs
    - https://www.youtube.com/watch?v=ygmd9ZWCb90
  - Trello for backlog and sprint planning
- Prepare project plan + feasibility study
- Choose database system
  - If MySQL, follow instructions https://github.com/JuhQ/Web-Programming-and-Data-Management/blob/main/Week9/database.md
- Create GitHub repository and add team + teachers as collaborators
- Bookmark and go through the CI/CD intro materials
  - CI/CD course content: https://fullstackopen.com/en/part11
- Time tracking sheet: https://docs.google.com/spreadsheets/d/1BM-cRwt8ZjfGlTmMNkl_oU6L9z8Va5L0LHiLaBsiMaM/edit?gid=0#gid=0
  - **MUST BE COPIED TO YOUR TEAM DRIVE, FILL IN EVERY DAY**

### Week 2

Focus:

- Begin coding
- Set up project workflow
- Start backend work
- Start database work, connect backend to database

Do this week:

- Start the codebase and connect app to database
- Everyone makes at least one git commit to the project and submits the commit link

### Week 3

Focus:

- Continue development
- Implement core features
- Learn practical Git workflow (branching, PRs, merge/rebase)

Do this week:

- Finalize team and topic
- Complete initial database implementation by end of week
- Keep steady weekly progress on features

### Week 4

Focus:

- Start CI/CD tasks from Full Stack Open Part 11
- Midterm demo of current project progress
  - CI/CD course content: https://fullstackopen.com/en/part11
  - Certificate info: https://fullstackopen.com/en/part0/general_info#course-certificate

Do this week:

- Begin CI/CD exercises
- Give the mandatory midterm demo (show current features, database status, and next steps)
- Work toward your personal certificate submission

### Weeks 4-7

Focus:

- Independent project development
- Mid-checkups and support sessions
- Project cleanup and documentation

Do during this period:

- Keep shipping features weekly
- Keep docs/README updated
- Prepare your presentation material

Hard deadlines:

- Week 4 (mid-week or end of week): mandatory midterm demo
- Week 5 (end of week): CI/CD task deadline
- Week 7 (end of week): presentation material submission

### Week 8

Focus:

- Finalize project and presentation
- Project presentations
- Final project submission

Do this week:

- Present your project clearly
- Presentation is mandatory for all team members
  - No-show = no grade for the project
- Submit final deliverables on time

## Grading Rubric

Final grade is based on overall project quality, teamwork process, and completed course requirements.

| Category                   | Weight | What We Look For                                                                |
| -------------------------- | ------ | ------------------------------------------------------------------------------- |
| Project functionality      | 50%    | Core features work, app is usable, key requirements are implemented             |
| Documentation              | 20%    | README is clear, setup steps work, project decisions are documented             |
| User experience and design | 15%    | App is intuitive, visually organized, and user-friendly                         |
| Team process and activity  | 15%    | Regular progress, checkup participation, daily time tracking, fair contribution |

### Minimum Requirements to Pass

- Frontend, backend, and database are connected and functional
- Attendance requirement (minimum 80%) is met
- Mandatory midterm demo (Week 4) is completed
- Team project is submitted on time
- Each student has visible personal contribution in Git history
- Required CI/CD certificate is submitted

### Grading Scale

- Grade 0: Pass requirements not met or major deliverables missing
- Grade 1-2: Basic implementation, limited scope or quality issues, but pass requirements met
- Grade 3-4: Good and reliable implementation with minor gaps
- Grade 5: Excellent technical quality, complete scope, polished documentation, strong teamwork
